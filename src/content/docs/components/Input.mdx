## Overview

Input components are essential UI elements that allow users to enter and edit text. They come in two main variants: single-line text inputs and multi-line text areas. Each type supports various states including basic, error, custom error messages, and disabled states.

## Single-line Text Inputs

### Basic Input

The basic text input includes a label, placeholder text, and optional helper text. The width will default to 100%, but can be set using postcss such as Tailwind as demonstrated here.

<ShowComponent>
  <BasicTextInput />
</ShowComponent>

### Error State

The error state indicates invalid input with red borders and an error message. It can be applied by changing the `aria-invalid` attribute to `true`.

<ShowComponent>
  <ErrorTextInput />
</ShowComponent>

### Custom Error Message

You can customize the error message to provide more specific feedback by adding the `input-error` class to the additional message. This will hide the helper text and show this instead, providing utility for cases where the helper text is insufficient (such as when a user name is already taken).

<ShowComponent>
  <CustomErrorTextInput />
</ShowComponent>

### Disabled State

The `disabled` attribute can be applied to the `<input/>` element to style the whole component to its disabled state.

<ShowComponent>
  <DisabledTextInput />
</ShowComponent>

### Clearable Input

Inputs can include a clear button to quickly remove entered text, although the functionality for this must be added in javascript.

<ShowComponent>
  <ClearTextInput />
</ShowComponent>

## Text Areas

### Basic Text Area

Text areas provide multi-line input capabilities with optional character limits. They behave in the same way as the `<input/>` above, but use a `<textarea/>` element instead.

<ShowComponent>
  <BasicTextArea />
</ShowComponent>

### Error State

Text areas support the same error states as single-line inputs. ike in the single line input, this is activated by changing the `aria-invalid` attribute to `true`.

<ShowComponent>
  <ErrorTextArea />
</ShowComponent>

### Custom Error Message

This includes the functionality of the custom error message.

<ShowComponent>
  <CustomErrorTextArea />
</ShowComponent>

### Disabled State

Text areas also support the disabled state. Like in the single line input, this is activated by applying the `disabled` attribute to the `<textarea/>`.

</ShowComponent>

## CSS Variables

The input system can be customized using the below CSS variables:

### General

#### Layout

| Variable                      | Default Value             | Description                   |
| ----------------------------- | ------------------------- | ----------------------------- |
| `--input-spacing`             | `0.25rem`                 | Gap between input elements    |
| `--input-padding-x`           | `0.5rem`                  | Horizontal padding            |
| `--input-padding-y`           | `0.5rem`                  | Vertical padding              |
| `--input-border-width`        | `1px`                     | Default border width          |
| `--input-border-width-active` | `2px`                     | Border width when active      |
| `--input-radius`              | `var(--radius-size-sm)`   | Border radius for inputs      |
| `--input-radius-textarea`     | `var(--radius-size-sm)`   | Border radius for textareas   |
| `--input-clear-radius`        | `var(--radius-size-full)` | Border radius of clear button |

#### Colors

| Variable                      | Default Value                     | Description                               |
| ----------------------------- | --------------------------------- | ----------------------------------------- |
| `--input-clear-surface`       | `var(--surface-controls-default)` | Background color of clear button          |
| `--input-clear-surface-hover` | `var(--surface-controls-dark)`    | Background color of clear button on hover |
| `--input-clear-color`         | `var(--white)`                    | Color of clear button icon                |

### Label Styling

#### Colors

| Variable                       | Default Value               | Description                   |
| ------------------------------ | --------------------------- | ----------------------------- |
| `--input-label-color`          | `var(--text-high-emphasis)` | Color of the input label      |
| `--input-label-color-disabled` | `var(--text-inactive)`      | Color of disabled label       |
| `--input-label-color-invalid`  | `var(--input-label-color)`  | Color of label in error state |

#### Typography

| Variable                        | Default Value                  | Description                         |
| ------------------------------- | ------------------------------ | ----------------------------------- |
| `--input-label-size`            | `var(--text-size-body)`        | Font size of the label              |
| `--input-label-line-height`     | `var(--line-height-body)`      | Line height of the label            |
| `--input-label-weight`          | `var(--font-weight-semi-bold)` | Font weight of the label            |
| `--input-label-weight-disabled` | `var(--input-label-weight)`    | Font weight of disabled label       |
| `--input-label-weight-invalid`  | `var(--input-label-weight)`    | Font weight of label in error state |
